# 🍬 キャンディラン (Candy Run)

お菓子いっぱいの迷路で、いもむしから逃げながらおやつを集める、**幼児向けのかわいい鬼ごっこゲーム**です。スマホでもPCでも、ブラウザだけで遊べます。

[Phaser 3](https://phaser.io/) 製。インストール不要・サーバー不要で動きます。

---

## 🎮 遊び方

パステルカラーのお菓子の世界の迷路で、**ピンクのうさぎ**を操作します。

- **動かす**
  - スマホ: 画面のどこでも指を置いてなぞる（バーチャルスティック）
  - PC: 矢印キー または WASD
- **目標**: 迷路に散らばったお菓子（キャンディ・ペロペロキャンディ・ドーナツ・グミ・マカロン）を、目標の数だけ集めると **レベルクリア**
- **敵**: いもむしがゆっくり追いかけてきます。捕まると **ハートが1つ減り**、少しの間むてきになって安全な場所へワープします。ハート3つが全部なくなると **ゲームオーバー**

### 難易度
- レベルが上がるほど、集めるお菓子が増え（15こ → 20こ … 最大30こ）、いもむしの数（最大3匹）とスピードも上がります
- 敵はゆっくりで、捕まっても即ゲームオーバーにはならないので、小さいお子さんでも遊べます

---

## ▶️ 遊ぶ / 動かす

### そのまま遊ぶ
`index.html` をブラウザで開くだけで遊べます（Phaser は CDN から読み込まれるため、インターネット接続が必要です）。

### ローカルサーバーで動かす（推奨）
ファイルを直接開くとブラウザによっては動かないことがあるため、簡易サーバー経由が確実です。

```bash
# Python がある場合
python3 -m http.server 8000
# → ブラウザで http://localhost:8000 を開く
```

## 📁 ファイル構成

```
candy-run/
├── index.html              エントリーポイント（画面・スクリプト読み込み）
├── assets/                 画像（PNG）。差し替え可能（下記参照）
└── js/
    ├── main.js             Phaser の起動・画面サイズ設定
    ├── MazeGenerator.js    迷路の自動生成
    ├── TextureFactory.js   画像が無いとき用の代替グラフィック生成
    ├── TouchControls.js    スマホ用バーチャルスティック
    ├── Player.js           主人公（移動・エフェクト）
    ├── Worm.js             いもむし（追いかけ・当たり判定）
    └── scenes/
        ├── TitleScene.js   タイトル画面
        ├── GameScene.js    ゲーム本編（迷路・お菓子・ライフ・スコア）
        └── ResultScene.js  クリア / ゲームオーバー画面
```

## 🎨 画像の差し替え

`assets/` に以下のファイル名で **背景透過PNG** を置くと、自動的に使われます。
無いものは、コード内で生成される図形（プレースホルダー）で動くので、一部だけの差し替えでもOKです。

| ファイル名 | 内容 | 向きの注意 |
|---|---|---|
| `player.png` | 主人公 | 右向き（左移動時は自動反転） |
| `worm_head.png` | いもむしの頭 | 上向き（進行方向に自動回転） |
| `worm_body.png` | いもむしの胴体1節 | 丸型なら向き不問 |
| `worm_tail.png` | いもむしのしっぽ | 上向き |
| `item1.png`〜`item5.png` | お菓子5種類 | 不問 |
| `wall.png` | 迷路の壁ブロック | 正方形 |
| `heart.png` | ライフのハート | 不問 |
| `logo.png` | タイトルロゴ | 横長（無ければテキスト表示） |

## 🛠 技術メモ

- **フレームワーク**: Phaser 3.60（CDN 読み込み。ビルド不要）
- **画面**: `Scale.RESIZE` で端末サイズいっぱいに表示。スマホの縦画面に対応
- **入力**: タッチ（バーチャルスティック）とキーボードの両対応
- **サウンド**: Web Audio API による手続き的なBGM・効果音（外部音源ファイル不要）
- **迷路**: 再帰的バックトラッキングで生成し、行き止まりを減らして遊びやすく調整
